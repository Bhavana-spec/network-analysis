# Packet Analysis

## baseline.pcap
- Captured normal HTTP and ICMP traffic between Kali and DVWA.
- Used as a reference for comparison with suspicious traffic.
- Wireshark filters used:
  - `http`
  - `icmp`

## nmap-scan.pcap
- Captured reconnaissance traffic generated by Nmap SYN scan.
- Observed SYN packets targeting the DVWA host.
- Wireshark filter used:
  - `tcp.flags.syn == 1 && tcp.flags.ack == 0`

## bruteforce.pcap
- Captured repeated POST requests related to login activity.
- Useful for identifying repeated authentication attempts.
- Wireshark filter used:
  - `http.request.method == "POST"`

## Suricata Validation
- Verified Suricata startup and log creation.
- Confirmed alert generation through `fast.log` and `eve.json`.
- Observed traffic visibility limitations in the desktop container environment.

## Summary
The lab successfully demonstrated:
- baseline traffic analysis
- reconnaissance traffic capture
- repeated login traffic generation
- IDS log validation
- packet-level investigation using Wireshark
